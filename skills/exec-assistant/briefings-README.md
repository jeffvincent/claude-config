# Weekly Briefings Archive

This document explains how to archive weekly leadership briefings generated by the executive assistant system. Actual briefings are saved in `resources/exec-assistant/briefings/` in the Knowledge System root.

## Purpose

The weekly briefing system synthesizes:
- Leadership priorities (from `resources/exec-assistant/priorities.md`)
- Calendar patterns and time allocation (from calendar analysis)
- Weekly management journal reflections (from `notes/management-journal/YYYY-MM.md`)
- Performance expectations and role alignment (from `resources/exec-assistant/expectations.md`)

Archiving these briefings allows you to:
- **Track evolution**: See how priorities and focus areas shift over time
- **Review progress**: Compare week-over-week changes in themes
- **Add reflections**: Annotate briefings with your own insights after the week concludes
- **Discuss with leadership coach**: Share specific briefings for deeper discussion
- **Git history**: See patterns in leadership focus through commit history

## File Naming Convention

**Format**: `YYYY-MM-DD.md`

Use the **Sunday date** that starts the week being briefed:
- `2026-03-02.md` = Week of March 2-8, 2026 (Sunday through Saturday)
- `2026-02-23.md` = Week of Feb 23-March 1, 2026

This matches the Sunday night timing when briefings are generated.

## Usage Workflow

### 1. Generate Briefing (Automated via Zapier)

**Zapier workflow** runs Sunday nights:
1. Fetches current week's management journal notes (`notes/management-journal/YYYY-MM.md`)
2. Fetches calendar data for the week
3. Runs Claude with `weekly-briefing-prep.md` prompt
4. Outputs formatted briefing

### 2. Save Briefing (Automated or Manual)

**Option A: Automated (Zapier → GitHub)**
- Zapier commits briefing directly to `resources/exec-assistant/briefings/`
- Creates file with proper date format
- Auto-commits to git with message

**Option B: Manual (from email)**
- Copy briefing content from email
- Create new file: `touch resources/exec-assistant/briefings/YYYY-MM-DD.md`
- Paste content and save
- Commit to git: `git add . && git commit -m "Add briefing for week of YYYY-MM-DD"`

### 3. Review and Annotate

After the week concludes, add your own reflections:

```markdown
## Week of March 2-8, 2026

[Generated briefing content]

---

## End-of-Week Reflection

**What landed:**
- [What actually got focus/traction this week]

**What shifted:**
- [Unexpected changes or pivots]

**Insights:**
- [Patterns or learnings from the week]

**For next week:**
- [Adjustments or carry-forward items]
```

### 4. Reference and Compare

Use git to track evolution:
```bash
# View recent briefings
ls -t resources/exec-assistant/briefings/*.md | head -5

# Compare two weeks
diff resources/exec-assistant/briefings/2026-02-23.md resources/exec-assistant/briefings/2026-03-02.md

# Search across all briefings
grep -r "delegation" resources/exec-assistant/briefings/

# View commit history of briefings
git log --oneline resources/exec-assistant/briefings/
```

## Zapier Integration Options

### Recommended: Zapier → GitHub Direct Commit

**Trigger**: Schedule (Sunday 9pm)

**Steps**:
1. **Code by Zapier**: Run briefing generation logic
   - Fetch management journal notes via GitHub API
   - Fetch calendar data via Google Calendar API
   - Call Claude API with weekly-briefing-prep.md prompt
2. **GitHub**: Create/Update File
   - Repository: `jeffvincent/knowledge-system`
   - Path: `resources/exec-assistant/briefings/YYYY-MM-DD.md`
   - Content: Formatted briefing output
   - Commit message: `Add weekly briefing for week of YYYY-MM-DD`

**Benefits**:
- Fully automated end-to-end
- Proper git history with timestamps
- No manual intervention needed

### Alternative: Zapier → Email → Manual Save

**Trigger**: Schedule (Sunday 9pm)

**Steps**:
1. Generate briefing (same as above)
2. Send email with briefing content
3. Manually copy from email to dated file in briefings folder
4. Commit to git

**Benefits**:
- Simpler Zapier setup (no GitHub integration needed)
- Still get email copy for mobile review
- Manual step allows for immediate annotations

## Git Workflow

This folder is **git-tracked** (part of Knowledge System repository).

**After adding new briefing:**
```bash
git add resources/exec-assistant/briefings/YYYY-MM-DD.md
git commit -m "Add weekly briefing for week of YYYY-MM-DD"
git push
```

**View history:**
```bash
git log --oneline resources/exec-assistant/briefings/
```

## Best Practices

1. **Use Sunday dates**: Briefings prepare for the coming week, so use the Sunday date that starts the week
2. **Add reflections**: After the week, annotate the briefing with what actually happened
3. **Review quarterly**: When running `/refresh-priorities`, review last 3 months of briefings to see patterns
4. **Share selectively**: Briefings contain sensitive strategic information - treat as private
5. **Track themes**: Use grep/search to find recurring themes across weeks
