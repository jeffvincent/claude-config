# Voice Authenticity Review — Pass/Fail Checklist

Run this checklist after every review draft. Every item must pass before delivering to the user.

## Preparation

- [ ] Read voice-patterns-v1.md completely (not skimmed)
- [ ] Analyzed ALL content (not just excerpts or first few paragraphs)

## Analysis Quality

- [ ] Identified issues at all three priority levels (High, Medium, Low)
- [ ] Every issue includes a specific location (paragraph/sentence number)
- [ ] Every issue cites a specific pattern from voice-patterns-v1.md
- [ ] Every issue includes a concrete rewrite suggestion (not just "fix this")
- [ ] No vague feedback — every observation is tied to a pattern

## Authentic Moments

- [ ] Identified 3-5 authentic moments (what works well)
- [ ] Each authentic moment explains WHY it works (pattern reference)

## Metrics

- [ ] Calculated average sentence length
- [ ] Calculated sentence length distribution
- [ ] Assessed vocabulary match percentage
- [ ] Included voice metrics table

## Deliverables

- [ ] Provided Quick Wins (top 3 changes for maximum impact)
- [ ] Included before/after examples for major issues
- [ ] Gave clear next steps with time estimates
- [ ] Stated confidence level (High/Medium/Low) with rationale

## Evolution

- [ ] Noted any new patterns not in voice-patterns-v1.md
- [ ] Offered to log insights to refinements tracker
