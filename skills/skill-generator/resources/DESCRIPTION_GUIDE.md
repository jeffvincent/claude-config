# Naming and Description Guide

## Limits
- Name: ≤ 64 characters
- Description: ≤ 200 characters

## What makes a good description
- States the job to be done AND when to use the Skill
- Uses clear, specific trigger language (e.g., "after X", "when drafting Y")
- Avoids vague benefits or marketing language

## Patterns
Good: "Extract action items and owners from meeting transcripts to create a follow‑up checklist. Use after a transcript is provided."

Weak: "Improve meeting notes."

Good: "Apply Acme brand guidelines (colors, fonts, logo usage) to slides and documents. Use when formatting external materials."

Weak: "Make docs on‑brand."
