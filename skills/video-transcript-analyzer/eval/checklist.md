# Video Transcript Analysis — Pass/Fail Checklist

Run this checklist after every analysis draft. Every item must pass before delivering.

## Format & Detection

- [ ] Correctly detected SRT vs plain text format
- [ ] If SRT, timestamps extracted and formatted accurately
- [ ] Video link included (if provided by user)
- [ ] Filename follows naming convention (see `instructions/naming-conventions.md`)

## Call Summary

- [ ] Summary is 2-3 paragraphs (not more, not less)
- [ ] Identifies who was on the call and their roles
- [ ] Covers main themes at high level
- [ ] Informative enough to skip the full transcript

## Key Quotes

- [ ] 5-8 quotes included
- [ ] ALL quotes are verbatim — zero paraphrasing
- [ ] Each quote has speaker attribution
- [ ] Quotes represent diverse themes (not all about one topic)
- [ ] Context provided where meaning isn't self-evident

## Topical Breakdown

- [ ] Topics are thematically organized (NOT just chronological)
- [ ] Each topic has a descriptive title (not "Topic 1" or "Issue A")
- [ ] Each topic has 3-6 detailed context bullets
- [ ] Timestamps are accurate (if SRT format)
- [ ] Related discussions are grouped even if they occurred at different times

## Transcript

- [ ] Full original transcript included at the end
- [ ] Transcript is unmodified — exact formatting preserved
- [ ] No content redacted unless explicitly requested

## Output Quality

- [ ] Single, well-formatted markdown document
- [ ] No customer information redacted (unless requested)
- [ ] Output written to correct path with correct filename
