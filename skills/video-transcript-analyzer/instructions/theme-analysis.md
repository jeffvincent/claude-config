# Theme Analysis Guidelines

Identify 5-10 major topics or themes from the conversation.

## What to Look For

- Pain points and problems mentioned
- Feature requests or desired functionality
- Workflow impacts and efficiency issues
- Perception or sentiment changes
- Specific examples or demonstrations
- Business impact statements

## Grouping Rules

- Group related discussions **thematically**, not just chronologically
- If the same topic comes up at 00:05:00 and again at 00:45:00, combine them under one theme
- Use descriptive topic headers that capture the substance, not generic labels
  - Good: "Membership Registration Email Management"
  - Bad: "Issue #1" or "Topic A"

## Per-Theme Structure

For each theme, provide:
- One-line description of what the section covers
- 3-6 detailed bullet points with specific context
- Include sub-issues, examples mentioned, and impact described
- For SRT: include the timestamp where the topic begins

## SRT Format
```markdown
**00:02:57 - Membership Registration Email Management**
Cannot bulk manage registration emails for large membership lists.

- Must manually resend registration emails one by one to 2,000+ members
- Search functionality constantly refreshes while typing
- Requires HubSpot support intervention for "hard resets"
- No ability to filter or create segments from membership data
```

## Plain Text Format
```markdown
## Topic #1: Membership Registration Email Management
Cannot bulk manage registration emails for large membership lists.

- Must manually resend registration emails one by one to 2,000+ members
- Search functionality constantly refreshes while typing
[same bullets]
```
